# 巴西经贸信息固定来源注册表

更新时间：2026-03-16

## 结论

建议采用“固定来源优先，广域搜索兜底”的工作流，而不是每次重新全网搜索。

原因很直接：

- 你要的内容本身偏官方、政策、数据，来源集中，重复搜索的收益很低。
- 自动化要稳定，最重要的是先固定一批高信噪比源，再做日期过滤、分类和去重。
- “本周无更新”本身也是有效结果，固定源模式更容易保留这类信息。

## 自动化建议

建议把每周流程固定成下面 6 步：

1. 固定源抓取
   - 先抓上面的一级来源，不做全网发散搜索。

2. 日期过滤
   - 严格限定到绝对日期区间，例如 `2026-03-09` 至 `2026-03-16`。

3. 主题过滤
   - 仅保留：经贸新闻、贸易数据、外贸制度、贸易政策、关税/反倾销/通关规则。
   - 排除：纯外交礼节、一般社会新闻、纯第三方评论。

4. 分类
   - 中方官方
   - 巴西官方
   - 第三方/其他公共来源
   - 本周无更新

5. 结构化输出
   - 日期
   - 标题
   - 来源机构
   - 来源类别
   - 主题标签
   - 原文链接
   - 对中国企业/研究用途的意义

6. 兜底搜索
   - 只有在固定源结果过少时，再做站外搜索。
   - 兜底搜索仍应优先 `site:gov.br`、`site:mofcom.gov.cn`、`site:br.mofcom.gov.cn`、`site:gov.cn`。

## 我对后续系统设计的专业建议

不建议以后每次“重新决定搜哪些网站”，建议直接把来源固定下来，然后每周只做：

- 抓最新
- 比日期
- 做分类
- 去重
- 输出周报

更稳妥的做法是把这套来源再沉淀成一个机器可读配置，例如 `sources.yaml`，字段至少包含：

- `name`
- `category`
- `country`
- `base_url`
- `list_url`
- `priority`
- `topics`
- `date_selector_or_rule`
- `enabled`

这样下一步就可以直接进入“半自动/全自动脚本化”阶段，而不是继续靠人工整理来源。
